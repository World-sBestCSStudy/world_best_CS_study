# 디자인 패턴

- 생성 패턴, 구조 패턴, 행위 패턴 별로 하나씩은 말할 수 있는게 좋다.
- 디자인 패턴이란 특정 역할의 모듈이나 모듈 사이의 인터페이스를 구현하는 코드 작성 시에 세부적인 구현 방식에 대한 대표적인 방식 또는 그 예시로,
특정한 상황에서 자주 발생하는 문제에 대한 재사용 가능한 솔루션을 제시하는 형식적인 방법이다.

### 디자인 패턴

- 싱글톤 패턴
- 팩토리 패턴
- 전략 패턴
- 옵저버 패턴
- 프록시 패턴과 프록시 서버
- 이터레이터 패턴
- 노출모듈 패턴
- MVC 패턴
- MVP 패턴
- MVVM 패턴
