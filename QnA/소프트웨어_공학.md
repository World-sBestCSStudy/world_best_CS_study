1. TDD에 대해 설명해주세요.
    ```
    TDD는 기존의 개발방식과는 다르게 먼저 테스트 코드를 작성하고, 그 테스트를 통과하기 위한 최소한의 코드를 작성하는 방식으로써  
    초기 비용이 비싸고 학습이 어렵다는 단점이 있지만 코드의 신뢰성을 높이고 버그를 초기 단계에서 쉽게 파악할 수 있다는 장점이 있습니다.
    ```
2. 클린코드에 대해 설명해주세요.
    ```
    클린코드란 의도와 목적이 명확하고 가독성이 좋은 코드를 말합니다.  
    비용과 개발시간을 효율적으로 사용하기 위해 적용하며, 코드를 표준화하고 단순화시켜 이해하기 좋게 만드는 것을 목적으로 합니다.  
    클린코드를 만드는 일반적인 방법으로는 네이밍 규칙 세우기, 적절한 주석 달기, 일관성있게 들여쓰기와 줄바꿈 하기 등이 있습니다.
    ```
3. 리팩토링에 대해 설명해주세요.
    ```
    가독성을 높이는 클린코드에 유지보수성 목적을 결합한 것으로, 외부 동작은 그대로 유지한 채 내부의 코드의 구조를 개선하는 것입니다.  
    리팩토링 작업은 코드를 이해하기 쉽고, 수정하기 쉽게 만들어주며, 향후 유지보수에 도움이 됩니다.
    ```
4. 리팩토링과 클린코드의 차이점을 말해보세요.
    ```
    클린 코드는 단순히 가독성을 높이기 위한 작업으로 이루어져 있다면,  
    리팩토링은 클린 코드를 포함한 유지보수를 위한 코드 개선이 이루어집니다.  
    클린코드와 같은 부분은 설계부터 잘 이루어져 있는 것이 중요하고,  
    리팩토링은 결과물이 나온 이후 수정이나 추가 작업이 진행될 때 개선해나가는 것이 올바른 방향입니다.
    ```
5. 애자일 개발 방법론에 대해 설명해주세요.
    ```
    기존의 개발 방식의 문제점을 해결하기 위해 등장한 다양한 해결책중 하나로,  
    기존의 개발 방식과 달리 문서보다 작동하는 소프트웨어를 중시하고, 이해관계자와 개발 과정을 함께함으로써 주기적으로 시제품을 보여주고 피드백을 받아 보완하는 방식으로 진행합니다.  
    주로 스프린트라는 주기를 가지고 진행하며 주기적인 회의인 스크럼 등을 활용합니다.
    ```
6. OOP에 대해 설명해주세요.
    ```
    OOP란 객체지향 프로그래밍을 말하며 크게 추상화, 캡슐화, 다형성, 상속이라는 특징을 갖습니다.  
    추상화란 객체에서 공통된 속성, 행위를 추출해 정의하는 것이고,  
    캡슐화는 서로 연관된 속성과 기능을 하나의 캡슐로 만들어 외부로부터 보호하는 것입니다.  
    다형성은 어떤 객체의 속성이나 기능이 여러가지 형태를 갖는 것입니다.  
    마지막으로 상속은 기존의 객체를 재활용하여 하위 객체를 생성하여 객체의 재사용성을 높인 것입니다.  
    객체지향 프로그래밍의설계 원칙으로는 SOLID라는 것이 있습니다.  
    또한 OOP는 재사용성과 확장성이 좋으며 유지보수가 편리하다는 장점이 있습니다.
    ```
7. 개발할 때 무엇을 고민하며 개발하는지 말해주세요.
    ```
    클린코드와 최적화된 코드를 지향합니다.  
    제가 생각하는 코드는 다른 팀원이 빠르게 이해할 수 있도록 명료한 코드이기에 불필요한 라인을 줄이고 필요한 부분엔 주석을 다는 방식으로 진행합니다.  
    또한 최적화를 고려하여 데이터베이스 통신을 줄일 수 있는 프록시 객체를 사용하고 N+1 문제를 줄이기 위해 FetchJoin이나 QueryDSL을 사용하는 방식을 채용하였습니다.
    ```
8. MSA에 대해 설명해주세요.
    ```
    MSA는 기존의 모놀리식 구조와 다르게 서비스들을 작게 분할하여 관리하는 방식으로  
    각 서비스는 특정 비즈니스 기능을 담당하며 독립적으로 개발되고 배포될 수 있습니다.  
    아키텍쳐 관리와 학습에 어려움이 있고  
    각 서비스들이 API통신으로 데이터를 주고 받기에 트래픽 부담이 있다는 단점이 있으나,  
    각 서비스를 독립적으로 개발 배포가 가능하여 유지보수에 용이하고,  
    한 서비스에 문제가 발생하였을 때 다른 서비스에 미치는 영향을 최소화할 수 있다는 장점이 있어  
    규모가 큰 서비스에서 많이 이용하고 있는 아키텍쳐입니다.
    ```
